# Rewind v0.1 — "It records" — design

Date: 2026-07-12
Status: approved (user delegated review)
Scope: ROADMAP milestone v0.1 only. Anything not needed for v0.1 is out.

## Goal

A user on macOS launches Rewind, a rolling replay buffer starts, and pressing
the global hotkey saves the last N seconds as an `.mp4` that appears in the
in-app clip library. Windows builds in CI from the same codebase (stub capture
until a Windows tester validates real capture).

## Success criteria

1. `flutter test` passes with **no native library required** (all Dart logic
   TDD'd against fakes).
2. `flutter run -d macos` works out of the box in **stub mode** (no libobs
   setup needed for contributors).
3. With libobs built via the fetch script, the app records real screen video:
   hotkey → `.mp4` of the last N seconds in `~/Movies/Rewind`, listed in the
   library with game/event/time/size.
4. CI: analyze + format + test + build macOS & Windows (stub mode) green.
5. Per-game buffer length applies when the active game changes (already-wired
   coordinator logic, now persisted and editable in Settings).

## Decisions (with alternatives considered)

### D1. Capture engine: dual-mode shim, script-built libobs

- The C shim keeps its **stub mode** as the default build so dev, tests, and
  CI need zero native setup.
- Real capture is compiled in when `REWIND_USE_LIBOBS` is defined.
  `tools/fetch_libobs.sh` clones `obs-studio` at a **pinned tag**, builds
  **libobs + only the plugin modules we need**, and caches the result under
  `native/third_party/obs/` (git-ignored). `hook/build.dart` detects that
  directory and adds the define + include/link flags.
- Plugin modules (macOS): `mac-capture` (ScreenCaptureKit `screen_capture`
  source, incl. system audio), `obs-ffmpeg` (muxer + `replay_buffer` output +
  VideoToolbox encoders), `coreaudio-encoder` (AAC). Windows (CI, later in the
  milestone): `win-capture`, `win-wasapi`, `obs-ffmpeg`, `obs-x264` fallback.
- Alternatives rejected: driving an installed OBS via obs-websocket (already
  rejected in ARCHITECTURE.md — external dependency, not self-contained);
  vendoring obs-studio as a submodule built on every compile (build times and
  dependency weight are hostile to contributors); shipping "prebuilt libobs"
  (no official SDK artifact exists for macOS — building once, pinned and
  cached, is the honest version of this).

### D2. `CaptureEngine` abstraction for TDD

`ClipCoordinator` and `main.dart` currently depend on the concrete FFI class.
Introduce:

```dart
abstract class CaptureEngine {
  bool init({required String outDir, required int seconds});
  bool startBuffer();
  bool stopBuffer();
  bool setBufferSeconds(int seconds);
  String? saveClip(String outDir);  // returns saved path or null
  void shutdown();
  String get lastError;
}
```

`RewindObsEngine` implements it over the existing `@Native` bindings;
`FakeCaptureEngine` (in `test/`) records calls and fabricates clip files in a
temp dir so coordinator/library/storage behavior is fully testable. This is
also the seam ARCHITECTURE.md reserves for a future alternate backend.

### D3. Persistence: two small JSON files

- **Settings** → `<app-support>/rewind/settings.json` (`AppSettings.toJson`
  already exists). Loaded at startup, saved on change. A `SettingsStore`
  takes an injected directory so tests use temp dirs.
- **Clip metadata** → `<clips-dir>/clips.json` maintained by `ClipLibrary`
  (add `toJson/fromJson` + load/save). On load, entries whose files no longer
  exist are dropped; files on disk without entries appear as `manual` clips
  (reconciliation keeps the library truthful if the user deletes files
  externally).
- No database. YAGNI — v0.3 storage policies can still be built on this.

### D4. Clips directory

`~/Movies/Rewind` on macOS, `%USERPROFILE%\Videos\Rewind` on Windows,
resolved through `path_provider` + `Platform`, created if missing, overridable
in settings later (not in v0.1 — YAGNI).

### D5. Hotkey + tray

- `HotkeyService` wraps `hotkey_manager`: parses the portable descriptor
  string (`"Alt+F10"`), registers the global hotkey, re-registers on change,
  invokes `ClipCoordinator.onHotkey`. Descriptor parsing is pure Dart → TDD.
- Tray (`tray_manager`): icon + menu — Open Rewind, Save clip now,
  buffer on/off toggle, Quit. Template (monochrome) icon on macOS.

### D6. UI ("modern gamer-dark, simple")

Three surfaces, custom dark theme (near-black surfaces, one electric accent,
generous spacing; no Material-default look, no RGB clutter):

1. **Home** — status strip: buffer state (recording dot + length), active
   game chip, "Save clip" button; below it the **clip library**: newest-first
   grid/list of clips with event badge, game, age, size; actions: play (open
   with OS default), reveal in Finder/Explorer, delete (with confirm).
2. **Settings** — hotkey recorder field, default buffer length
   (30 s / 60 s / custom), per-game list with per-game buffer override.
3. **Tray menu** — quick actions (D5).

Widget tests cover: library renders clips from a `ClipLibrary`, empty state,
settings edits round-trip through `SettingsStore`, status strip reflects
coordinator state.

### D7. Real shim implementation (macOS bring-up)

Behind `REWIND_USE_LIBOBS`, implement in `rewind_obs.c`:

- `rewind_obs_init`: `obs_startup` → `obs_reset_video/audio` (display
  resolution, 60 fps) → load required modules → create `screen_capture`
  source → VideoToolbox H.264 encoder (+ AAC) → `replay_buffer` output with
  `max_time_sec = seconds`.
- `rewind_start_buffer` / `rewind_stop_buffer`: start/stop the output.
- `rewind_save_clip`: call the output's `"save"` proc handler, wait for the
  `"saved"` signal (bounded wait), read `"get_last_replay"` path.
- `rewind_set_buffer_seconds`: `obs_output_update` with new `max_time_sec`.
- Every failure path sets `rewind_last_error`.

Screen-recording permission (macOS TCC) is requested on first init; the app
surfaces a clear error state in the UI when permission is missing.

### D8. CI

Keep the existing `ci.yml` shape (stub-mode builds on both OSes — fast).
Add one macOS job that runs `tools/fetch_libobs.sh` (cached by pinned tag) and
builds the app with real libobs to prove linkage doesn't rot. `flutter create`
runner directories (`macos/`, `windows/`) are committed — today they don't
exist, so **no build currently works**; this is task 0.

## Error handling

- Shim not built / init fails → app runs in "capture unavailable" state: UI
  banner with the `rewind_last_error` text; hotkey/tray disabled but library
  still browses existing clips.
- Save fails (returns null/error) → non-blocking toast; nothing added to
  library.
- macOS screen-recording permission missing → explicit banner with a button
  opening System Settings → Privacy → Screen Recording.
- Settings/clips JSON corrupt → back up the bad file, start from defaults
  (never crash on parse).

## Out of scope (YAGNI, deferred to later milestones)

League watcher changes (v0.2), StorageManager policies/pinning (v0.3),
trim/export/share (v0.5), signing/notarization/auto-update (v1.0),
custom clips-dir setting, thumbnails/video preview in library (placeholder
tiles in v0.1).

## Compliance note

v0.1 touches no game data source at all — only screen capture via libobs and
the manual hotkey, which `docs/COMPLIANCE.md` classifies as universally safe.
The `GameEventSource` seam stays untouched so v0.2 games plug in without core
changes.
